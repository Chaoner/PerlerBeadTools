# 拼豆图片生成网站项目需求文档

版本：0.1  
日期：2026-05-09  
阶段：需求确认，不进入开发

## 1. 项目目标

制作一个网页工具，让用户上传图片后，将图片转换成可实际制作的拼豆图纸。网站需要输出像素化预览、真实品牌色号匹配、每种颜色珠子数量、拼板分区、可打印图纸和可导出的材料清单。

核心原则：

- 生成结果必须“能摆出来”，不是只做视觉滤镜。
- 默认规则贴近市场常用 5mm Midi 拼豆和 29x29 标准方形拼板。
- 高级用户可以切换到 2.6mm Mini 拼豆，获得更高细节。
- 所有颜色匹配以真实可买到的品牌色板为目标，而不是任意 RGB 色。

## 2. 市场规则调研结论

### 2.1 常见拼豆尺寸

市场主流热熔拼豆通常分为三档：

| 规格 | 常见直径 | 适用场景 | 本项目定位 |
| --- | ---: | --- | --- |
| Mini | 约 2.5mm / 2.6mm | 肖像、复杂插画、高细节作品 | 高级模式 |
| Midi / Standard | 约 5mm | 入门、角色、头像、常规图案 | 默认模式 |
| Maxi / Biggie | 约 10mm | 儿童大颗粒、安全易摆放 | 后续可选，不做 MVP 默认 |

依据：

- Hama 官方说明 Mini 为 2.5mm，Midi 为 5mm，Maxi 为 10mm。
- Perler 官方 1000 颗单色包标注实际珠子尺寸约为 5.07mm 高 x 4.77mm 宽。
- Artkal 官方产品页长期区分 2.6mm、5mm、10mm 三类拼豆。

### 2.2 常见拼板规则

本项目默认以正方形网格生成图纸。

默认拼板规则：

- 标准 5mm 模式：1 张基础大方板按 29x29 珠位处理，共 841 颗。
- 多板作品：按 29 的倍数分区，例如 58x58、87x87、116x116。
- Mini 模式：保留同样的“按珠位计数”逻辑，但物理尺寸按 2.6mm 估算。
- 网格坐标：使用行列坐标，便于用户按图摆放。

说明：Perler 官方大方形透明拼板标注为 5.7 英寸见方，并支持多张拼接；按 5mm 珠距换算，接近 29x29 珠位。市场零售页也常直接写为 29x29。

### 2.3 色板规则

首版应该支持三类色板：

| 色板 | 用途 | 首版策略 |
| --- | --- | --- |
| Perler 5mm | 北美市场常见，适合标准拼豆 | 内置基础色板，可后续补全 |
| Hama Midi/Mini | 欧洲及通用拼豆市场常见 | 内置基础色板，可后续补全 |
| Artkal 5mm / 2.6mm | 颜色数量多，适合高还原度作品 | 作为高级色板目标 |

颜色匹配要求：

- 不直接使用图片原始 RGB 颜色作为最终色。
- 每个像素必须映射到所选品牌色板中的一个色号。
- 颜色距离优先使用感知色差算法，例如 Lab 色彩空间 + CIEDE2000 或近似 Delta E。
- 用户可以限制最大颜色数，例如 8、16、24、32、64 色。
- 用户可以排除特殊材质色，例如透明、夜光、珠光、荧光色。

## 3. 用户与使用场景

目标用户：

- 拼豆爱好者：想把头像、动漫角色、宠物、游戏图标转成图纸。
- 家长或老师：想快速生成适合儿童制作的简单图案。
- 手作卖家：想估算材料数量、尺寸、色号，方便备货。
- 像素艺术用户：想把图片转成标准网格作品。

典型流程：

1. 上传图片。
2. 裁剪或选择保留比例。
3. 选择拼豆尺寸、品牌色板、目标宽高或拼板数量。
4. 调整生成模式和颜色数量。
5. 预览生成效果。
6. 手动修补局部颜色。
7. 导出图纸和材料清单。

## 4. MVP 功能范围

### 4.1 图片上传

必须支持：

- 上传 JPG、PNG、WEBP。
- 拖拽上传和点击选择文件。
- 显示原图预览。
- 显示图片尺寸和文件大小。

限制建议：

- 单张图片不超过 20MB。
- 前端本地处理，默认不上传服务器。
- 图片透明区域可识别为“空珠位”或指定背景色。

### 4.2 图像裁剪与尺寸设置

必须支持：

- 保持原比例。
- 裁剪成目标比例。
- 填充背景。
- 指定目标宽度和高度，单位为“颗珠”。
- 快速尺寸预设：
  - 29x29：单板小图案。
  - 58x58：2x2 拼板。
  - 87x87：3x3 拼板。
  - 116x116：4x4 拼板。
  - 自定义尺寸：10 到 200 颗宽，高度按比例或手动设置。

生成时显示：

- 珠子总数。
- 预计物理尺寸。
- 需要几张 29x29 拼板。
- 每张拼板的坐标范围。

### 4.3 生成模式

必须支持三种模式：

| 模式 | 目标 | 处理倾向 |
| --- | --- | --- |
| 简洁图标 | 动漫角色、Logo、表情包 | 强化边缘，减少颜色，保留轮廓 |
| 肖像模式 | 人像、宠物 | 保留肤色层次，适度平滑噪点 |
| 真实还原 | 风景、复杂照片 | 更多颜色，更少边缘强化 |

核心算法流程：

1. 读取图片并应用裁剪。
2. 缩放到目标珠位网格。
3. 根据模式做降噪、锐化、对比度和饱和度调整。
4. 颜色量化到限制色数。
5. 将每个格子匹配到品牌色板。
6. 可选抖动，用于改善渐变，但默认关闭，避免实物出现噪点。
7. 输出网格数据、颜色统计和预览图。

### 4.4 预览

必须支持：

- 原图与生成图并排对比。
- 像素方块预览。
- 圆珠预览。
- 中空圆珠预览，模拟真实拼豆孔洞。
- 显示或隐藏网格线。
- 放大、缩小、适配屏幕。
- 鼠标悬停显示行列坐标、色号、颜色名。

### 4.5 手动编辑

必须支持：

- 画笔：替换单个或多个珠位颜色。
- 橡皮擦：把珠位设为空。
- 吸管：从图纸中取色。
- 油漆桶：区域填充。
- 替换颜色：将某个色号全部替换为另一个色号。
- 撤销和重做。

### 4.6 材料统计

必须输出：

- 总珠子数量。
- 每个颜色的数量、品牌、色号、颜色名、色块。
- 按数量从多到少排序。
- 估算需要购买的 1000 颗单色包数量。
- 拼板数量，例如 2x2 张、3x3 张。

可选：

- 用户输入已有库存后，显示缺口数量。

### 4.7 导出

必须支持：

- PNG 预览图导出。
- PDF 图纸导出。
- CSV 材料清单导出。

PDF 内容要求：

- 封面：作品名称、尺寸、总珠数、色数、拼板数。
- 图纸页：按 29x29 拼板分页。
- 每页显示行列坐标。
- 每个格子可以显示颜色、色号缩写或符号。
- 材料清单页：完整颜色统计。

后续增强：

- SVG 导出。
- 1:1 实物尺寸打印。
- 镜像导出，用于胶带法或反面摆放。

## 5. 拼豆生成规则详述

### 5.1 默认生成规则

首版默认值：

- 拼豆规格：5mm Midi / Standard。
- 拼板单位：29x29。
- 默认输出尺寸：58x58。
- 默认色板：Perler 基础色板。
- 默认最大颜色数：32。
- 默认抖动：关闭。
- 默认背景：透明区域为空珠位，非透明背景保留。

### 5.2 物理尺寸估算

计算规则：

- 宽度 mm = 珠位宽度 x 单颗直径。
- 高度 mm = 珠位高度 x 单颗直径。
- 5mm 模式下，58x58 约为 290mm x 290mm。
- 2.6mm 模式下，58x58 约为 150.8mm x 150.8mm。

说明：实际尺寸会受到品牌、熔融程度和拼板间距影响，网站只显示“估算尺寸”。

### 5.3 拼板分区

按 29x29 切分：

- 每个区块称为一个 Board。
- Board 命名规则：A1、A2、B1、B2。
- 每页导出一个 Board 或多个 Board。
- 边界线需要明显，方便用户拼接。

示例：

- 29x29 = 1 张板。
- 58x58 = 4 张板。
- 87x58 = 6 张板。
- 100x100 = ceil(100/29) x ceil(100/29) = 16 张板。

### 5.4 空珠位规则

透明 PNG 或被用户擦除的区域作为空珠位处理。

空珠位：

- 不计入总珠数。
- 导出时显示为空白或斜线。
- 不参与颜色统计。

### 5.5 色差与替换规则

颜色匹配：

- 使用品牌色板中的 RGB/HEX 值。
- 将颜色转换到 Lab 空间后计算色差。
- 选择色差最小的色号。

颜色压缩：

- 当用户限制最大颜色数时，先对图片做颜色聚类，再映射到色板。
- 若多个聚类映射到同一色号，应合并统计。

替代色：

- 每个颜色可展示 1 到 3 个近似替代色。
- 替代色用于缺货或库存不足场景。

## 6. 页面与交互需求

### 6.1 首屏

首屏应该直接是工具，不做营销落地页。

必须包含：

- 上传区。
- 原图预览。
- 生成结果预览。
- 右侧或底部设置面板。
- 导出按钮。

### 6.2 设置面板

包含：

- 拼豆规格：5mm、2.6mm。
- 品牌色板：Perler、Hama、Artkal、自定义。
- 宽高设置。
- 最大颜色数。
- 生成模式。
- 抖动开关。
- 背景处理。
- 预览样式。

### 6.3 编辑工作区

包含：

- 工具栏：画笔、橡皮擦、吸管、填充、撤销、重做、缩放。
- 色板栏：当前使用颜色。
- 统计栏：总珠数、颜色数、拼板数。

## 7. 非功能需求

### 7.1 隐私

- MVP 默认在浏览器本地处理图片。
- 不上传用户图片到服务器。
- 导出文件在本地生成。

### 7.2 性能

目标：

- 58x58 图纸生成时间小于 1 秒。
- 116x116 图纸生成时间小于 3 秒。
- 200x200 图纸生成时间小于 8 秒。
- 大图处理不能卡死界面，必要时使用 Web Worker。

### 7.3 浏览器支持

优先支持：

- Chrome 最新版。
- Edge 最新版。
- Safari 近两个大版本。
- 移动端浏览器可查看和轻量编辑，但复杂编辑优先桌面体验。

### 7.4 可访问性

- 所有按钮有明确标签。
- 色块不能只依赖颜色区分，导出图纸需提供符号或色号。
- 支持键盘撤销、重做、缩放。

## 8. 数据结构建议

### 8.1 图纸数据

```ts
type Pattern = {
  id: string;
  name: string;
  beadSizeMm: 5 | 2.6 | 10;
  brand: "perler" | "hama" | "artkal" | "custom";
  width: number;
  height: number;
  cells: PatternCell[];
  palette: BeadColor[];
  createdAt: string;
};
```

### 8.2 珠位数据

```ts
type PatternCell = {
  x: number;
  y: number;
  colorId: string | null;
};
```

### 8.3 色号数据

```ts
type BeadColor = {
  id: string;
  brand: string;
  code: string;
  name: string;
  hex: string;
  finish?: "solid" | "transparent" | "glow" | "pearl" | "neon";
};
```

## 9. MVP 不包含内容

首版暂不做：

- 用户账号系统。
- 云端保存。
- 在线支付或材料购买。
- 社区图纸库。
- AI 文生图。
- 自动背景抠除。
- 移动端高级绘图体验。

这些可以作为二期功能。

## 10. 验收标准

MVP 完成后应满足：

- 用户可以上传一张图片并生成拼豆图纸。
- 用户可以选择 5mm 或 2.6mm 规格。
- 用户可以选择至少一个内置品牌色板。
- 用户可以设置目标宽高和最大颜色数。
- 生成结果只使用所选色板中的颜色。
- 可以显示总珠数、每色数量和拼板数量。
- 可以手动修改珠位颜色。
- 可以导出 PNG、PDF 和 CSV。
- 58x58 默认图纸生成流畅。
- 图片默认在本地处理，不上传服务器。

## 11. 推荐技术方向

后续开发建议：

- 前端框架：React + TypeScript + Vite。
- 图像处理：Canvas API + Web Worker。
- PDF 导出：pdf-lib 或 jsPDF。
- CSV 导出：原生 Blob。
- 状态管理：Zustand 或轻量 React state。
- 本地保存：IndexedDB。

理由：

- 这是偏交互和图像处理的单页工具，前端本地完成最合适。
- TypeScript 有利于维护图纸、色板、导出等结构化数据。
- Web Worker 可以避免大图处理时界面卡顿。
